# brand-report

שלב 5: מדידה, דוח שבועי, ואופטימיזציה מתמשכת.

## Usage
`/brand-report [week N|compare]`

- *(ללא ארגומנט)* — דוח לשבוע הנוכחי
- `week N` — דוח לשבוע ספציפי
- `compare` — השוואה בין שני שבועות אחרונים

---

## תהליך

### שלב א: איסוף נתונים
הפעל את `analytics-agent` לאיסוף/בקשת נתוני ביצוע (תנועה, לידים, רכישות, הוצאה, הכנסה) מהמשתמש אם אינם ב-memory.

### שלב ב: חישוב מדדים והפקת דוח
`analytics-agent` מפיק את הדוח השבועי המלא (ראה `analytics-agent.md`, סעיף 3), כולל אבחון (סעיף 4).

### שלב ג: המרת תובנות לפעולה
לפי האבחון, נתב אוטומטית:
```
בעיית creative/הוק → copywriting-agent + design-agent
בעיית דף נחיתה → web-dev-agent
בעיית טרגוט/תקציב → growth-marketing-agent
בעיית איכות ליד (מגיע אך לא נסגר) → sales-agent
ROAS מעולה ויציב → growth-marketing-agent (סקייל)
```

### שלב ד: מצב `compare`
הצג טבלת השוואה בין שני השבועות האחרונים (הוצאה/הכנסה/ROAS/רכישות/CPA) ומסקנה.

### שלב ה: עדכון Roadmap
עדכן `brand-director` בסטטוס הכולל ובפעולות שנפתחו למחלקות אחרות כתוצאה מהדוח.

הצג סיכום עם המלצה ברורה לשבוע הבא.
